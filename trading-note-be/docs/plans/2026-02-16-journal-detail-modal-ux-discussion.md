**Date**: 2026-02-16
**Participants**:
- **Alex Kim** - Lead Product Designer (Fintech/Trading UX 12y)
- **Yuna Park** - Interaction Designer (Mobile-first patterns, micro-interactions 8y)
- **Daniel Choi** - Visual Designer (Data visualization, dark UI specialist 10y)
- **Miso Lee** - UX Researcher (Trading platform usability studies 6y)

## Round 1: Current State Analysis

**Miso Lee (UX Researcher)**:
> 현재 모달을 분석하면 핵심 문제가 3가지입니다.
> 1. **정보 계층이 평면적** - P&L 히어로, 진입가/청산가, 거래 정보, 수익률 분석이 모두 같은 시각적 무게로 나열되어 있음. 사용자가 "이 거래를 빠르게 복기"하려면 가장 중요한 정보가 먼저 눈에 들어와야 하는데, 스크롤해야 전체를 볼 수 있음.
> 2. **이미지 영역 미노출** - chartScreenshotUrl이 있어도 모달에서 눈에 잘 안 띔. 트레이더에게 차트 이미지는 "그 순간의 맥락"을 담는 가장 강력한 정보인데, 작은 영역에 밀려 있음.
> 3. **액션 접근성** - 수정/삭제 버튼이 헤더 우상단에 작게 있어서 의도적으로 찾아야 함.

**Alex Kim (Product Designer)**:
> 동의합니다. 트레이딩 저널의 본질은 "복기"입니다. 이 모달을 열었을 때 트레이더가 얻어야 하는 것은:
> 1. 이 거래가 이익이었나 손실이었나 (즉시 판단)
> 2. 왜 이 진입을 했는가 (차트 이미지 + 메모)
> 3. 숫자적 결과 (P&L, ROI, 투자금)
>
> 현재 레이아웃은 숫자를 먼저 보여주고 맥락(이미지, 메모)은 아래로 밀려 있습니다.
> **차트 이미지와 P&L을 나란히 배치하는 2-column 히어로**가 더 효과적일 것입니다.

**Alex Kim**:

### Proposal B: Stacked with Image First (Yuna Park)

```
+--------------------------------------------------+
```

**Yuna Park (Interaction Designer)**:
> 모바일에서는 2-column이 깨집니다. Stacked 방식이 반응형에 유리합니다. 대신 이미지를 모달 상단, 헤더 바로 아래에 크게 배치하면 "이 거래의 맥락"이 즉시 전달됩니다. 이미지가 없으면 P&L 히어로가 바로 보이고요.

### Proposal C: Hybrid - Desktop Split, Mobile Stacked (Daniel Choi)

**Daniel Choi**:
> 두 제안을 합치면 됩니다. Desktop(sm 이상)에서는 Split Hero, 모바일에서는 Stacked. Tailwind의 반응형 클래스로 쉽게 구현 가능합니다.
>
> 추가로 **색상 시스템 개선**이 필요합니다:
> - 이익 거래: 전체적으로 emerald tint가 미묘하게 깔림
> - 손실 거래: red/rose tint
> - 현재 bg-emerald-900/20 같은 배경이 너무 약합니다. 좀 더 명확한 시각적 단서가 필요합니다.

## Round 3: Image Display Deep Dive

**Miso Lee**:
> 트레이딩 플랫폼 사용자 조사 결과, 차트 스크린샷을 활용하는 트레이더의 87%가 "차트를 보면서 메모를 읽고 싶다"고 응답했습니다. 이미지와 메모가 동시에 보이는 것이 중요합니다.

**Alex Kim**:
> 이미지 관련 결정사항:
> 1. **이미지가 있으면**: 모달 상단에 큰 프리뷰 표시 (aspect-ratio 16:9 권장)
> 2. **클릭하면**: 전체 화면 오버레이로 확대 (현재 구현과 동일)
> 3. **이미지가 없으면**: 해당 섹션을 완전히 숨기고 P&L 히어로가 올라옴

**Daniel Choi**:
> 이미지 프리뷰 디자인:
> - rounded-xl, overflow-hidden
> - 이미지 위에 반투명 그래디언트 오버레이 + 심볼/P&L 텍스트 올려놓기 (이미지와 핵심 정보를 동시에)
> - 이건 너무 복잡할 수 있습니다. 단순하게 이미지 아래에 P&L을 배치하는 게 더 깔끔합니다.

**Yuna Park**:
> 오버레이 텍스트는 이미지 가독성을 해칩니다. **이미지는 이미지 자체로, P&L은 별도 섹션으로** 분리하는 게 맞습니다. 단, 이미지 영역을 현재 h-48보다 더 크게 (h-56~64) 잡아야 합니다.

### Team Consensus on Images:
- 이미지 있을 때: 헤더 바로 아래, `aspect-video` (16:9) 크기로 표시
- object-cover가 아닌 object-contain + 어두운 배경으로 이미지 왜곡 방지
- 클릭 시 풀스크린 확대 유지
- 이미지 없을 때: 섹션 완전 숨김

## Round 4: Information Architecture Refinement

**Alex Kim**:
> 정보 계층을 재정리합시다. 중요도순:
>
> **Tier 1 (즉시 인지)**: P&L 금액, ROI%, 이익/손실 상태
> **Tier 2 (핵심 데이터)**: 진입가, 청산가, 수량, 투자금, 차트 이미지
> **Tier 3 (부가 정보)**: 거래 유형, 자산 유형, 레버리지, 시드 비율
> **Tier 4 (분석/복기)**: 메모, 리스크 관리, 수익률 프로그레스 바

**Yuna Park**:
> Tier 3 정보(거래 유형 등)는 별도 섹션이 아니라 헤더의 배지 시스템으로 충분합니다. 현재 이미 Symbol 옆에 현물/롱/숏 배지가 있으니, 추가 "거래 정보" 그리드는 과잉입니다. 핵심 정보만 남기면 스크롤 없이 한 화면에 다 보입니다.

**Daniel Choi**:
> 동의합니다. "거래 정보" 섹션을 제거하고 헤더 배지로 대체하면:
> - 자산 유형 + 거래 유형: 헤더 배지
> - 화폐: KRW 기본이면 생략 가능
> - 포지션/레버리지: 헤더 배지에 이미 있음
> - 시드 비율: 수익률 분석 섹션에 통합

**Miso Lee**:
> "수익률 분석" 프로그레스 바도 재고해봅시다. 2.64% ROI를 100% 기준 프로그레스 바로 보여주면 거의 안 보입니다. 프로그레스 바 대신 **숫자 자체를 크고 명확하게** 보여주는 것이 더 효과적입니다.

### Team Consensus on Layout:
1. **Header**: Symbol + 배지(현물/선물, LONG/SHORT) + 날짜 + 액션 버튼
2. **Image Section** (조건부): 차트 스크린샷 큰 프리뷰
3. **P&L Hero**: 손익 금액 + ROI (크고 명확하게)
4. **Metrics Grid**: 진입가, 청산가, 수량, 투자금 (4열 그리드)
5. **Risk Management** (조건부): 손절가, 익절가, R:R, 최대 손실
6. **Memo Section** (조건부): 트레이딩 메모
7. **Footer**: 결과 복사 + 이전/다음 네비게이션

## Round 5: Micro-Interactions & Polish

**Yuna Park**:
> 세부 인터랙션 제안:
> 1. **P&L 금액에 tabular-nums** - 숫자 정렬 일관성
> 2. **이미지 hover시 확대 아이콘 오버레이** - 클릭 가능함을 시각적으로 알림
> 3. **네비게이션 화살표 키** - 이미 구현됨, 유지
> 4. **메모 영역**: 배경색을 amber에서 slate 톤으로 변경. amber가 "경고"로 오인될 수 있음

**Daniel Choi**:
> 색상 개선:
> - 메모 배경: `bg-slate-100 dark:bg-slate-800/50` (중립적 톤)
> - P&L 히어로 배경: 이익시 `bg-emerald-50 dark:bg-emerald-950/30`, 손실시 `bg-red-50 dark:bg-red-950/30`
> - 프로그레스 바 제거, 대신 ROI를 P&L 바로 아래에 크게 표시
> - Metrics 카드: 현재 `bg-slate-100 dark:bg-slate-800/50` 유지하되 패딩 약간 조정

**Alex Kim**:
> 최종적으로 "거래 정보" 섹션과 "수익률 분석" 프로그레스 바를 제거하면, 대부분의 거래가 **스크롤 없이 한 화면**에 보입니다. 이것이 가장 큰 UX 개선입니다.

## Final Design Decision

### Layout (Top to Bottom):

```
+--------------------------------------------------+
| [Symbol] [현물] [주식]    2025.01.15  [Edit][Del][X]|
+--------------------------------------------------+
| [Chart Screenshot - aspect-video, object-contain]  |
| (이미지 없으면 이 섹션 자체가 사라짐)                  |
| (hover 시 확대 아이콘 표시)                          |
+--------------------------------------------------+
|              +528,000원                            |
|              +2.64%                                |
|  (emerald/red 배경 tint, 아이콘 포함)               |
+--------------------------------------------------+
|  진입가      |  청산가      |  수량    |  투자금     |
|  220,000원   |  237,600원   |  30주    | 6,600,000원|
+--------------------------------------------------+
|  [리스크 관리 - 조건부]                              |
|  손절가 | 익절가 | R:R | 최대 손실                   |
+--------------------------------------------------+
|  [메모 - 조건부]                                    |
|  slate 톤 배경, 깔끔한 텍스트                        |
+--------------------------------------------------+
|  결과 복사           [이전]  1/10  [다음]            |
+--------------------------------------------------+
```

### Key Changes from Current:
1. **이미지를 상단으로 이동** - P&L 위에 배치, 큰 크기
2. **"거래 정보" 섹션 제거** - 헤더 배지로 충분
3. **"수익률 분석" 프로그레스 바 제거** - P&L 히어로의 ROI가 충분
4. **메모 배경색 amber -> slate** - 중립적 톤
5. **전체적으로 컴팩트** - 스크롤 최소화
6. **이미지 hover 시 확대 아이콘** - 클릭 어포던스

### Rejected Ideas:
- 이미지 위 오버레이 텍스트 (가독성 문제)
- Desktop 2-column split (이미지 없을 때 어색, 구현 복잡도 대비 효과 미미)
- 탭 UI로 정보 분리 (한 화면에 다 보이는 게 낫다)
